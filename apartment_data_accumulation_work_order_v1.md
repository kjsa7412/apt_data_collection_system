# 아파트 분석 서비스 데이터 적재 작업지시서 v1

## 문서 목적

본 문서는 아파트 분석 서비스 구축을 위해 지금까지 정의한 요소 마스터를 **순차적으로 누적 구축**하기 위한 실행용 작업지시서다.  
목표는 무작정 데이터를 늘리는 것이 아니라, **서비스 목적에 맞는 순서로 마스터를 정비하고, 각 단계별 산출물이 다음 단계의 입력이 되도록 구조화**하는 것이다.

이 문서는 아래 목적을 가진다.

1. 앞으로 쌓아야 할 데이터 영역을 우선순위 순서로 정리한다.
2. 각 단계에서 어떤 데이터를 어떤 목적에 맞게 뽑아야 하는지 명확히 한다.
3. 모든 단계가 **엑셀 적재 가능**, **DB 적재 가능**, **점수화 가능**, **코드화 가능**한 형태로 작성되도록 통제한다.
4. 실제 작업자가 각 차수별로 그대로 복사해 사용할 수 있는 프롬프트를 제공한다.

---

# 1. 추진 원칙

## 1-1. 기본 원칙

- 한 번에 모든 데이터를 수집하지 않는다.
- **마스터 정의 → 정량 데이터 구조화 → 정성 데이터 구조화 → 코드 표준화 → 점수화 → 컷오프/리스크** 순으로 간다.
- 모든 단계의 출력은 **엑셀에 바로 붙여넣을 수 있는 탭 구분 텍스트**를 원칙으로 한다.
- 데이터는 설명용이 아니라 **운영용 마스터**로 작성한다.
- 각 단계의 결과는 반드시 다음 단계에서 재사용될 수 있어야 한다.
- 같은 개념을 여러 표현으로 반복하지 않고, 서비스 관점에서 통합 가능한 것은 통합한다.
- 정량/정성/코드/불리언/계산값을 명확히 구분한다.
- MVP에서 실제 수집 가능한 데이터부터 우선적으로 쌓는다.

## 1-2. 누적 구축 방식

향후 데이터 적재는 아래 순서를 따른다.

1. 공통 적재 규칙을 먼저 고정한다.
2. 서비스의 평가 엔진에 직접 연결되는 기본 마스터부터 구축한다.
3. 이후 세부 마스터를 확장한다.
4. 마지막에 코드 표준화, 점수화, 컷오프 체계를 정리한다.
5. 각 단계 결과를 기준으로 실제 수집 우선순위를 정한다.

# 3. 공통 상위 프롬프트

아래 프롬프트는 이후 모든 단계 프롬프트에 공통으로 앞에 붙여 사용한다.

```text
당신은 대한민국 아파트 시세·평가·급지·미래가치 분석 서비스를 위한 데이터 설계 담당자다.

목표는 특정 주제의 요소를 서비스에서 누적 가능한 “마스터 데이터” 형태로 정리하는 것이다.
단순 설명문이 아니라, 이후 DB 적재, 엑셀 관리, 점수화, 코드화, 컷오프 판정까지 연결될 수 있도록 구조적으로 정리해야 한다.

절대 규칙:
1. 출력은 엑셀 붙여넣기 가능한 탭 구분 텍스트로 작성한다.
2. 헤더를 반드시 포함한다.
3. 동일 개념의 중복 표현은 정리하고, 서비스 관점에서 통합 가능한 것은 통합한다.
4. 정량/정성/코드/불리언 여부를 명확히 구분한다.
5. “설명용”이 아니라 “운영용 마스터”가 되도록 작성한다.
6. 지금 단계에서 필요한 수준보다 과도하게 복잡한 계산식은 넣지 말고, 운영 가능한 범위로 정리한다.
7. 항목은 한국 아파트 시장 기준으로 현실적으로 자주 쓰이는 것부터 우선 정리한다.
8. 출력 항목 수가 많아도 되며, 누적 확장이 가능하도록 빠짐없이 쓴다.

기본 출력 컬럼:
대분류 / 요소 / 세부항목 / 데이터유형 / 단위/코드예시 / 세부기준/범위 예시 / 영향 / 비고

추가 지시:
- 가능한 경우 코드값 예시를 함께 제시한다.
- 가능한 경우 구간값 예시를 함께 제시한다.
- 서비스에서 바로 쓸 수 있도록 명칭은 일관되게 정리한다.
- 결과는 markdown 표가 아니라 탭 구분 텍스트로 출력한다.
```

# 4. 단계별 실행 프롬프트

## Phase 0. 공통 적재 규칙 확정 프롬프트

```text
아파트 분석 서비스의 모든 마스터 데이터를 앞으로 일관되게 누적하기 위한 “공통 적재 규칙 마스터”를 작성하라.

목적:
- 이후 작성될 모든 마스터가 같은 구조를 사용하도록 기준을 고정하는 것
- 데이터유형, 코드값, 정량값, 정성값, 불리언값, 계산값의 구분 원칙을 명확히 하는 것
- 마스터 데이터 관리자가 이후 항목을 계속 추가하더라도 구조가 흔들리지 않게 하는 것

반드시 포함할 것:
1. 공통 컬럼 정의
2. 컬럼별 작성 원칙
3. 데이터유형 구분 기준
4. 코드값 작성 규칙
5. 구간값 작성 규칙
6. 정성값 작성 규칙
7. 계산값 필드 사용 규칙
8. 중복 항목 통합 원칙
9. 향후 마스터 확장 시 체크리스트
10. 금지 표현 및 비권장 표현

출력 형식:
엑셀 붙여넣기 가능한 탭 구분 텍스트
헤더 포함
```

## Phase 1. 단지 시설/스펙 마스터 프롬프트

```text
아파트 분석 서비스에서 단지 자체의 상품성을 평가하기 위한 “단지 시설/스펙 마스터”를 작성하라.

- 단지명만으로는 알 수 없는 실제 상품성 차이를 데이터화하는 것

반드시 포함할 것:
1. 난방 방식
2. 복도식/계단식/혼합형
3. 총세대수
4. 동수
5. 최고층
6. 준공연도 및 경과연수
7. 주차대수
8. 지하주차장 연결성
9. 엘리베이터 관련 항목
10. 커뮤니티 시설 항목
11. 보안/출입통제 항목
12. 택배/무인보관/생활서비스 관련 항목
13. 쓰레기 처리 방식
14. 외관/조경/관리 상태 관련 항목
15. 리모델링/시설 개선 여부
16. 실거주 체감에 직접 영향을 주는 세부 항목

출력 형식:
엑셀 붙여넣기 가능한 탭 구분 텍스트
헤더 포함
컬럼은 공통 마스터 형식을 따른다
```


# 7. 최종 요약

본 작업은 “데이터를 많이 모으는 것”이 목적이 아니라,  
**아파트 분석 서비스에 필요한 데이터를 평가 가능하고 누적 가능하며 운영 가능한 구조로 만드는 것**이 목적이다.

따라서 향후 적재는 아래 원칙을 유지해야 한다.

- 먼저 기준을 정하고
- 그 다음 핵심 정량 데이터를 쌓고
- 이후 정성 데이터를 구조화하며
- 마지막에 코드화, 점수화, 컷오프화로 이어간다

이 문서는 앞으로 이어질 모든 데이터 적재 작업의 기준 문서로 사용한다.
